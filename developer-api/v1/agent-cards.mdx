## What are Agent Cards?

Agent Cards let your AI agents make real purchases using Ramp. When an agent needs to buy something, it generates a unique payment credential through Visa Intelligent Commerce (VIC) — scoped to the specific merchant and capped at the requested transaction amount. Every transaction shows up in your Ramp audit log attributed to the agent on behalf of the authorized user.

Agent Cards are accessible to anyone in your org with access to Ramp [funds](https://support.ramp.com/hc/en-us/articles/10881975647763-Funds-cards-and-spend-programs). The [Ramp CLI](/developer-api/v1/cli) respects existing permissions, so users can only issue cards against funds they're scoped to.

"Buy this SaaS license for the team — here's the pricing page"
"Purchase 50 units of office supplies from this Amazon listing"

### Install the CLI

```bash Install with the one-liner
curl -fsSL https://agents.ramp.com/install.sh | sh
```

Or clone the repo directly:

```bash Clone the CLI repo
git clone https://github.com/ramp-public/ramp-cli.git
```

### Authenticate

```bash Log in to Ramp
ramp auth login
```

This opens a browser-based OAuth flow. For more details, see the [CLI guide](/developer-api/v1/cli#authentication).

### Install the agentic-purchase skill

Give your agent the skill URL so it knows how to use Agent Cards:

```text
https://github.com/ramp-public/ramp-cli/blob/main/src/ramp_cli/skills/agentic-purchase/SKILL.md
```

The repo includes starter code for browser-based purchasing, but Agent Cards work with any agent framework or workflow. Use them however fits your setup.

## Your First Agent Transaction

Follow the walkthrough in the CLI repo's `/skills` folder to make your first agent purchase. The flow works like this:

1. Your agent identifies something to purchase
2. It requests a payment credential scoped to the merchant and amount
3. The credential is used to complete the transaction
4. The purchase appears in your Ramp audit log, attributed to the agent

## Authorization & Sessions

<Info>
## Session expiry

Read-only sessions expire **1 week** after last usage. Read-write sessions expire **24 hours** after last usage.
</Info>

To enable the CLI on a remote machine (e.g., a cloud-hosted agent):

```bash Copy credentials to a remote machine
# 1. Authenticate locally
ramp auth login


# 2. Copy the config to the remote machine
scp ~/.config/ramp/config.toml remote-host:~/.config/ramp/config.toml
```

As long as the agent makes at least one request within the refresh window (24h for read-write, 1w for read-only), the session will not expire.

## Current Limitations

We're working with Visa to reduce or remove these limitations as soon as possible. If any of these block your workflow, reach out and we'll do our best to unblock you.

### Spend limit

$5K/day issuance limit per business. If this blocks your use case, reach out and we can consider raising this limit on a case-by-case basis.

### 3D Secure (3DS) support

Visa is actively working to support [3DS](https://en.wikipedia.org/wiki/3-D_Secure) via VIC. 3DS is commonly used in non-U.S. markets to reduce fraud and protect merchants from liability in the event of fraud, and is primarily required for online transactions within the European Economic Area (EEA) & UK, India, Japan, Australia, and some other countries.

### Merchant-initiated transaction (MIT) / Card on File (CoF) support

Visa is actively working to support MIT/CoF transactions via VIC, which enables merchants to charge your card asynchronously. This payment processing flow is common for subscriptions (e.g. SaaS, memberships), booking travel, recurring billing (e.g. utilities, insurance), and metered billing (e.g. cloud services, token-based AI spend).

We're actively tuning the experience during alpha. Please report:
