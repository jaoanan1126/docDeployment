This guide will help you make your first Ramp API call in just a few minutes. You'll create an access token and call the Transactions API to retrieve transaction data.

This guide assumes you are a Ramp customer with **admin** access to your dashboard. Need a test environment for development? Request access to our [Sandbox](/developer-api/v1/sandbox).

## 1. Create your app and get credentials

First, you'll need to create a developer app to get your API credentials.

<Steps>
<Step title=" ">
In your Ramp account, go to **Company → Developer** (you'll need admin access).
</Step>
<Step title=" ">
Click **Create New App**.
</Step>
<Step title=" ">
Name your app (e.g., "API Quickstart") and accept the Terms.
</Step>
<Step title=" ">
Under **Grant types**, click **Add new grant type** → select **Client Credentials**.
</Step>
<Step title=" ">
Under **Scopes**, click **Configure allowed scopes** and select `transactions:read`.
</Step>
<Step title=" ">
Copy your **Client ID** and **Client Secret**. Store them securely – you'll need them in the next step.
</Step>
</Steps>

For more details on authorization, see the [Authorization Guide](/developer-api/v1/authorization).

## 2. Get an access token

With your credentials, request an access token. Ramp authenticates this `/developer/v1/token` request with HTTP Basic Auth: base64-encode `client_id:client_secret` and send it in the `Authorization` header. The token authenticates your API requests.

Replace `{environment-url}` with:

- `https://demo-api.ramp.com` for Sandbox
- `https://api.ramp.com` for Production

<Info>
**API URLs are different from the Ramp app URLs**

The API base URL is not the same as the URL for the Ramp customer app. Make sure you're pointing your API calls at the API host, not the frontend host.

| Environment | API base URL | Ramp app URL |
| --- | --- | --- |
| Production | `https://api.ramp.com` | `https://app.ramp.com` |
| Sandbox (demo) | `https://demo-api.ramp.com` | `https://demo.ramp.com` |
</Info>

<CodeGroup dropdown>
```bash Request an access token
curl -X POST '{environment-url}/developer/v1/token' \
  -H "Authorization: Basic $(echo -n "$CLIENT_ID:$CLIENT_SECRET" | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "scope=transactions:read"
```
```javascript Request an access token
const response = await fetch('{environment-url}/developer/v1/token', {
  method: 'POST',
  headers: {
    'Authorization': `Basic ${btoa(`${clientId}:${clientSecret}`)}`,
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: new URLSearchParams({
    grant_type: 'client_credentials',
    scope: 'transactions:read'
  })
});


const data = await response.json();
console.log(data);
```
```python Request an access token
import base64, requests


auth = base64.b64encode(f"{client_id}:{client_secret}".encode()).decode()


response = requests.post(
  "{environment-url}/developer/v1/token",
  headers={
    "Authorization": f"Basic {auth}",
    "Content-Type": "application/x-www-form-urlencoded"
  },
  data={
    "grant_type": "client_credentials",
    "scope": "transactions:read"
  }
)


print(response.json())
```
</CodeGroup>

Response:

```json
{
  "access_token": "ramp_tok_abc123...",
  "token_type": "bearer",
  "expires_in": 864000,
  "scope": "transactions:read"
}
```

Copy the `access_token` – you'll use it to make API calls. Tokens expire after 10 days.

## 3. Make your first API call

With your access token, you can now call the Ramp API. Let's retrieve your transactions using the [GET /transactions](/api-reference/transaction/list-transactions#get-developer-v1-transactions) endpoint.

Add the header `Authorization: Bearer <YOUR_TOKEN_HERE>` and make a GET request to `{environment-url}/developer/v1/transactions`.

<CodeGroup dropdown>
```bash Get transactions
curl -X GET '{environment-url}/developer/v1/transactions' \
  -H "Authorization: Bearer $ACCESS_TOKEN"
```
```javascript Get transactions
const response = await fetch('{environment-url}/developer/v1/transactions', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});


const data = await response.json();
console.log(data);
```
```python Get transactions
response = requests.get(
  "{environment-url}/developer/v1/transactions",
  headers={"Authorization": f"Bearer {access_token}"}
)


print(response.json())
```
</CodeGroup>
