Ramp Embedded Cards allow you to securely display card numbers and CVV codes to users via an iframe. If you want to programmatically create cards for a virtual card issuing flow where partners can easily create virtual cards to pay travel, maintenance, and other third-party vendors on their own platforms, see our [Virtual Cards guide](/developer-api/v1/virtual-cards).

This guide walks you through embedding Ramp card details in your application. Card numbers and CVV codes are securely displayed to your users via an iframe—card data flows directly from Ramp to your user's browser without touching your servers.

**Prerequisites:** This guide assumes you have already created a fund and have the `card_id` of a card you want to display to your end users, and that you have a domain where your application is hosted (`parent_origin`). For information on creating funds and issuing cards, see our [Cards and Funds guide](/developer-api/v1/cards-and-funds). For more on `parent_origin`, see the [parent_origin section](#what-is-parent_origin) below.

<Warning>
Security Notice

Never store tokens client-side. All tokens should be securely stored on your backend server.
</Warning>

## How It Works

This integration follows a four-step flow:

1. **Create a limit** → Create a limit using [`POST /limits`](/api-reference/limit/create-a-limit#post-developer-v1-limits-deferred). A card will be automatically created and each card has a unique `card_id`
2. **Backend generates embed token** → Your backend calls `/developer/v1/embedded/cards/{card_id}/embed` to generate a secure embed token
3. **Frontend loads iframe** → Your frontend receives the token and loads it in an iframe pointing to Ramp's embed service
4. **User views card** → The user's browser requests card details directly from Ramp—card data never touches your servers

## Step 1: Enable the embedded_cards Scope

Before you can generate embed tokens, you need to enable the `embedded_cards:write` scope on your application.

### Demo Environment

Enable the `embedded_cards:write` scope on your app in the [developer dashboard](https://demo.ramp.com/settings/ramp-developer).

## Step 2: Create Endpoint to Generate Embed Token

Create an API endpoint that accepts a `card_id` and `parent_origin`, then calls Ramp's API (`POST /embedded/cards/{card_id}/embed`) to generate an embed token.

**Request body:**

```json
{
  "parent_origin": "your_parent_origin"
}
```

```python Python/Flask Example
from flask import Flask, request, jsonify
import requests
import os


app = Flask(__name__)


RAMP_API_BASE = "https://demo-api.ramp.com/developer/v1"
RAMP_ACCESS_TOKEN = os.environ["RAMP_ACCESS_TOKEN"]


@app.route("/api/generate-embed-token", methods=["POST"])
def generate_embed_token():
    data = request.json
    card_id = data.get("card_id")
    parent_origin = data.get("parent_origin")
    
    response = requests.post(
        f"{RAMP_API_BASE}/embedded/cards/{card_id}/embed",
        headers={"Authorization": f"Bearer {RAMP_ACCESS_TOKEN}"},
        json={"parent_origin": parent_origin}
    )
    
    if response.status_code == 200:
        return jsonify(response.json())
    else:
        return jsonify({"error": "Failed to generate token"}), response.status_code


if __name__ == "__main__":
    app.run(port=3001)
```

```javascript Node.js/Express Example
const express = require('express');
const axios = require('axios');


const app = express();
app.use(express.json());


const RAMP_API_BASE = 'https://demo-api.ramp.com/developer/v1';
const RAMP_ACCESS_TOKEN = process.env.RAMP_ACCESS_TOKEN;


app.post('/api/generate-embed-token', async (req, res) => {
  const { card_id, parent_origin } = req.body;
  
  try {
    const response = await axios.post(
      `${RAMP_API_BASE}/embedded/cards/${card_id}/embed`,
      { parent_origin },
      {
        headers: {
          'Authorization': `Bearer ${RAMP_ACCESS_TOKEN}`
        }
      }
    );
    
    res.json(response.data);
  } catch (error) {
    res.status(error.response?.status || 500).json({
      error: 'Failed to generate token'
    });
  }
});


app.listen(3001, () => {
  console.log('Server running on port 3001');
});
```

**Response:**

```json
{
  "embed_init_token": "{{embed_init_token}}"
}
```

## Step 3: Implement Frontend

Your frontend calls your backend endpoint to get the embed token, then loads the iframe.

### React Implementation

```jsx RampEmbeddedCard.jsx
import React, { useEffect, useRef } from 'react';


const RampEmbeddedCard = ({ cardId }) => {
  const iframeRef = useRef(null);


  useEffect(() => {
    const loadCard = async () => {
      // Call your backend to get embed token
      const response = await fetch('/api/generate-embed-token', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          card_id: cardId,
          parent_origin: window.location.origin
        })
      });


      const data = await response.json();
      const embedToken = data.embed_init_token;


      // Load iframe
      const iframe = iframeRef.current;
      if (iframe) {
        iframe.src = 'https://demo-embed.ramp.com';
        
        iframe.onload = () => {
          setTimeout(() => {
            iframe.contentWindow.postMessage(
              {
                type: 'ramp.embed.init',
                token: embedToken
              },
              'https://demo-embed.ramp.com'
            );
          }, 100);
        };
      }
    };


    loadCard();
  }, [cardId]);


  return (
    <iframe
      ref={iframeRef}
      title="Ramp Card"
      style={{ width: '100%', height: '400px', border: 'none' }}
      allow="clipboard-write"
    />
  );
};


export default RampEmbeddedCard;
```

### Using the Component

```jsx App.jsx
import RampEmbeddedCard from './RampEmbeddedCard';


function App() {
  return (
    <div>
      <h1>My Card Details</h1>
      <RampEmbeddedCard cardId="a40a6ce8-70d4-4d06-91e1-0728ad9bbe39" />
    </div>
  );
}


export default App;
```

## What is parent_origin?

The `parent_origin` is the domain where your application is hosted. This must exactly match the `window.location.origin` of the page where you'll embed the iframe.

This is a critical security parameter—Ramp validates that the embed request is being loaded from the specified origin, preventing unauthorized domains from displaying your users' card details.

**Examples:**

- Production: `"https://your-app-domain.com"`
- Local development: `"http://localhost:3000"`

The origin must match exactly—including protocol (`https://` vs `http://`) and port number. Mismatched origins will result in the embed failing to load.

#### How do I get a card_id?

You can retrieve card IDs via the [Limits API](/api-reference/limit/list-limits) or when creating cards through the [Virtual Cards guide](/developer-api/v1/virtual-cards).

#### Can I use this in production?

Yes, but you'll need to [contact developer support](https://app.ramp.com/chat?entryVariant=developerSupport) to enable the `embedded_cards:write` scope for your production application.

#### What if the parent_origin doesn't match?

The embed will fail to load if the `parent_origin` doesn't exactly match the origin of the page loading the iframe. Make sure to include the protocol and port number.
