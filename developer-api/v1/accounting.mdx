Ramp's API can be used to keep data in sync between Ramp and an external accounting system.

## Building an Accounting Integration

This guide is targeted towards developers building a connection between Ramp and a remote accounting/ERP system.

By following this guide, you'll enable seamless data flow between Ramp and your ERP, keeping vendors, GL accounts, and transaction data synchronized.

### Implementation Checklist

Building an accounting integration involves three main steps: configuring the connection, uploading your chart of accounts, and implementing object sync.

Configure: Establish the accounting connection

Upload: Import chart of accounts from your ERP

Sync: Fetch objects from Ramp and mark as synced

## Create Accounting Connection

Before syncing your chart of accounts, you need to establish an accounting connection between Ramp and your ERP system. This connection serves as the foundation for all accounting data synchronization.

<video controls>
  <source src="https://docs.ramp.com/assets/Building%20an%20accounting%20integration%20-%20Getting%20started-vqSjz8LN.mp4" />
</video>

**Overview of accounting connections in Ramp**

An Accounting Connection is the link between your Ramp account and your company's accounting system (ERP). It allows Ramp to exchange data with your ERP so that transactions, vendors, accounts, and rules stay in sync.

Each accounting connection is uniquely associated with its own set of:

- Accounting Custom Fields
- Accounting Vendors
- Accounting GL Accounts
- Accounting Rules

For more information, [learn about accounting configurations in Ramp](https://support.ramp.com/hc/en-us/sections/4454478746387-Accounting).

**Managing Accounting Connections**

A single Ramp account can only have one active accounting connection at a time.

Switching the accounting connection will result in a loss of data from these fields and rules, but it is possible to re-activate a connection which will restore them.

**Check Connection Status**

**Accounting Connection Types**

There are three types of accounting connections:

### Direct Connection

Managed by Ramp for out-of-the-box integrations

- Activated when customers onboard to QuickBooks, NetSuite, etc.
- Ramp-managed setup and maintenance
- Can only be enabled in the Ramp UI (requires authorization)
- Can be disabled via API or Ramp UI
- When enabled, Custom Fields, Vendors, and GL Accounts become read-only via API

[Overview of Ramp Accounting](https://support.ramp.com/hc/en-us/articles/4434982407443-Overview-of-Ramp-Accounting)

### Universal CSV

File-based import/export for quick setup

- File-based import and export
- API compatible with special UI behavior
- Quick setup for immediate use
- See Universal CSV Overview guide

[Universal CSV Overview](https://support.ramp.com/hc/en-us/articles/4463895129875-Universal-CSV-overview)

### Developer API

Full API control with advanced features

- Complete API read/write access
- Optimized for passive sync patterns
- Latest features and capabilities
- Recommended for custom integrations

**Multi-Entity Connections**

A single Ramp account can only have one active accounting connection at a time, however Ramp supports multi-entity connections: one NetSuite (or any other kind of accounting connection) account with multiple subsidiaries.

Each subsidiary has its own chart of accounts and sync settings. For example, one NetSuite account with multiple subsidiaries (e.g., TechCorp parent with CloudSoft, DataSys, and MobilePro subsidiaries) is supported by Ramp.

Switching the accounting connection will result in a loss of data from these fields and rules, but it is possible to re-activate a connection which will restore them.

**Migrating Connections**

<Warning>
**Migration will result in data loss**

Migrating Accounting Connections will result in a loss of historical chart of accounts codings. Ensure chart of accounts is backed up before proceeding with a migration.
</Warning>

Two accounting connections cannot be active at once. When building an accounting connection to an external system, it is highly recommended to disconnect the existing connection and establish a new one. In case things go wrong, you can always reconnect the previous connection and regain access to the old Custom Fields, Vendors, and GL accounts.

Before disconnecting, it is recommended to fetch all of the existing configuration and store it server-side so that the accounting structure can be rebuilt with the new accounting connection. See more guidance on how to handle migrations in the [Migration section](#migration).

#### View Current Accounting Connection

You can check if an accounting connection already exists by calling the [GET /developer/v1/accounting/connection](/api-reference/accounting/fetch-the-current-active-accounting-connection#get-developer-v1-accounting-connection) endpoint:

```bash
curl \
  -H "Accept: application/json" \
  -H "Authorization: Bearer $RAMP_API_TOKEN" \
  'https://api.ramp.com/developer/v1/accounting/connection'
```

#### Disconnect Accounting Connection

If you need to disconnect an existing accounting connection, use the [DELETE /developer/v1/accounting/connection](/api-reference/accounting/disconnect-an-accounting-connection#delete-developer-v1-accounting-connection) endpoint:

```bash
curl \
  -X DELETE \
  -H "Accept: application/json" \
  -H "Authorization: Bearer $RAMP_API_TOKEN" \
  'https://api.ramp.com/developer/v1/accounting/connection'
```

<Warning>
**Disconnecting will clear accounting data**

Disconnecting an accounting connection will remove all associated custom fields, GL accounts, and vendors. See the [Migration section](#migration) for guidance on preserving data during connection changes.
</Warning>

#### Create New Accounting Connection

To create a new Developer API accounting connection, use the [POST /developer/v1/accounting/connection](/api-reference/accounting/register-a-new-api-based-accounting-connection#post-developer-v1-accounting-connection) endpoint:

```bash
curl \
  -X POST \
  -H "Accept: application/json" \
  -H "Authorization: Bearer $RAMP_API_TOKEN" \
  -d '{
    "remote_provider_name": "ACCOUNTING_SEED"
  }' \
  'https://api.ramp.com/developer/v1/accounting/connection'
```

The `remote_provider_name` field identifies your accounting system.

## Chart of Accounts and Custom Fields

When integrating your ERP with Ramp, one of the first setup steps is syncing your chart of accounts — the list of general ledger accounts, vendors, and classification fields that ensure every Ramp transaction is properly coded in your ERP.

This section walks through how to:

- Understand what's imported into Ramp from your ERP
- Upload data for each category (GL Accounts, Custom Fields, Vendors)
- Verify the data inside the Ramp UI after import

#### Chart of Accounts Overview

| **ERP Object** | **Required?** | **Configuration** | **Description** | **Ramp API Endpoint** |
|---|---|---|---|---|
| **GL Accounts** | ✅ Always required | API configurable | Primary accounting categories (e.g., expense or revenue accounts). | [POST /developer/v1/accounting/accounts](/api-reference/accounting/upload-general-ledger-accounts#post-developer-v1-accounting-accounts) |
| **Custom Fields** | ⚙️ Optional | API configurable | Adds metadata such as department or cost center. | [POST /developer/v1/accounting/fields](/api-reference/accounting/create-a-new-custom-accounting-field#post-developer-v1-accounting-fields) |
| **Custom Field Options** | ⚙️ Optional | API configurable | Options for custom fields (e.g., department values). | [POST /developer/v1/accounting/field-options](/api-reference/accounting/upload-new-options#post-developer-v1-accounting-field-options) |
| **Vendors** | ⚙️ Optional | API configurable | Defines supplier names for bills or transactions. | [POST /developer/v1/accounting/vendors](/api-reference/accounting/upload-vendors#post-developer-v1-accounting-vendors) |
| **Entities** | ✅ Always required | UI only (read-only via API) | Business units or subsidiaries configured directly in Ramp. | UI only |

<Warning>
**Chart of Accounts cannot be modified for Ramp accounts using a direct accounting connection**

This applies to GL accounts, custom fields, and vendors - they become read-only via API when a direct connection is active.
</Warning>

#### Uploading GL Accounts

GL Accounts define the categories under which your company records transactions — such as expenses, revenue, or assets. These accounts are imported from your ERP and uploaded to Ramp to ensure consistent coding across systems.

**Example GL accounts:**

| **Classification** | **Code (optional)** | **ID** | **Name** |
|---|---|---|---|
| EXPENSE | 6410 | 514 | Employees: Salaries & Wages |
| EXPENSE | — | 418 | Office/Admin: Phone & Internet |
| REVENUE | — | 425 | Sales |

To upload your GL accounts to Ramp, use the [POST /developer/v1/accounting/accounts](/api-reference/accounting/update-a-general-ledger-account#patch-developer-v1-accounting-accounts-gl-account-id) endpoint:

```bash
curl \
  -X POST \
  -H "Accept: application/json" \
  -H "Authorization: Bearer $RAMP_API_TOKEN" \
  -d '{"gl_accounts":[{"classification":"EXPENSE","code":"6410","id":"514","name":"Employees:Salaries & Wages"},{"classification":"EXPENSE","id":"418","name":"Office/Admin:Phone & Internet"},{"classification":"REVENUE","id":"425","name":"Sales"}]}' \
  'https://api.ramp.com/developer/v1/accounting/accounts'
```

#### Creating Custom Fields and Adding Options

Custom fields let you import classifications from your ERP like Department, Cost Center, or Location.

**Example Custom Field:**

| **Field Name** | **Input Type** |
|---|---|
| Department | SINGLE_CHOICE |

To create a new custom field in Ramp, use the [POST /developer/v1/accounting/fields](/api-reference/accounting/create-a-new-custom-accounting-field#post-developer-v1-accounting-fields) endpoint:

```bash
curl \
  -X POST \
  -H "Accept: application/json" \
  -H "Authorization: Bearer $RAMP_API_TOKEN" \
  -d '{"id":"Department","input_type":"SINGLE_CHOICE","is_splittable":"True","name":"Department"}' \
  'https://api.ramp.com/developer/v1/accounting/fields'
```
