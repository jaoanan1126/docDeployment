### Overview

This guide explains how integrators can enable and use vendor credits with the Ramp API.

<Warning>
Note: This feature is currently in Beta.
</Warning>

### Setup Guide and Feature Support

**Requirements**

- Your app must have scopes `accounting:read`, `accounting:write`, and `vendors:read`

### Step 1. Enable vendor credits on your accounting connection

<Info>
Note

This setting is managed at the Accounting Connection level. If you plan on creating multiple Accounting Connections, you need to configure this setting for each connection.
</Info>

**For new connections:** Make a `POST` call to [/developer/v1/accounting/connection](/api-reference/accounting/register-a-new-api-based-accounting-connection#post-developer-v1-accounting-connection) with `settings.vendor_credits_enabled` set to `true`:

```bash
curl -X POST "{environment-url}/developer/v1/accounting/connection" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "is_active": true,
    "remote_provider_name": "Your ERP Name",
    "settings": {
      "vendor_credits_enabled": true
    }
  }'
```

**For existing connections:** First, retrieve your connection ID by making a `GET` call to [/developer/v1/accounting/connection](/api-reference/accounting/fetch-all-accounting-connections-for-the-current-business#get-developer-v1-accounting-all-connections):

```json
{
  "connection_type": "API",
  "is_active": true,
  "status": "linked",
  "settings": {
    "vendor_credits_enabled": true
  },
  "id": "8c0955d6-c8bf-4421-b3b5-d450b91afa7b",
  "last_linked_at": "2025-11-02T21:29:19+00:00",
  "created_at": "2025-11-02T21:29:19+00:00",
  "remote_provider_name": "ACCOUNTING_SEED"
}
```

Then make a `PATCH` call to [`/developer/v1/accounting/connection/{connection_id}`](/api-reference/accounting/update-an-accounting-connection#patch-developer-v1-accounting-connection-connection-id) with the `id` to enable vendor credits:

```bash
curl -X PATCH "{environment-url}/developer/v1/accounting/connection/<connection_id>" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "settings": {
      "vendor_credits_enabled": true
    }
  }'
```

### Step 2. Create vendor credits in the Ramp UI

Once vendor credits are enabled, users can create them in the Ramp platform:

1. Navigate to **Vendors**
2. Click into a specific vendor
3. Click **New** in the right-hand corner
4. Select **Vendor Credit** and fill in the required fields

![Creating a vendor credit in Ramp](https://docs.ramp.com/assets/vendor-credits-BMiUiQyS.png)

```bash
curl -X GET "{environment-url}/developer/v1/vendors/credits" \
  -H "Authorization: Bearer <token>"
```

**List vendor credits for a specific vendor**

To retrieve all vendor credits associated with a specific vendor, make a `GET` call to [`/developer/v1/vendors/{vendor_id}/credits`](/api-reference/vendor/list-vendor-credits-by-vendor#get-developer-v1-vendors-vendor-id-credits). Use the `vendor_id` from the response above or from your vendor records:

```bash
curl -X GET "{environment-url}/developer/v1/vendors/<vendor_id>/credits" \
  -H "Authorization: Bearer <token>"
```

This endpoint returns the same response format as the general credits list, but filtered to only include credits for the specified vendor.
