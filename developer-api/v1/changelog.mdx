- **Draft user status support** ŌĆö Added `USER_DRAFT` enum value to the user `status` filter and response field, along with a new `scheduled_invitation_date` field on user responses. [View endpoint](/developer-api/v1/api/users#get-developer-v1-users)

- **Business balance structured amounts** ŌĆö The business balance response now includes `CurrencyAmount` fields (e.g., `available_card_limit_amount`, `card_limit_amount`, `statement_balance_amount`) with explicit currency codes and minor unit values alongside the existing numeric fields. [View endpoint](/developer-api/v1/api/business#get-developer-v1-business-balance)

- **Reimbursement trace ID** ŌĆö Reimbursement detail responses now include a `trace_id` field with the bank-facing accountability reference for the payment.

- **New limit state** ŌĆö Added `ARCHIVED` enum value to the limit `state` field.

- **New currency support** ŌĆö Added `USDT` to supported currency codes across bill, purchase order, and other endpoints.

- **New accounting field type** ŌĆö Added `UNIT_OF_MEASURE` to the `remote_type` enum for custom accounting fields.

- **New webhook event types** ŌĆö Added `bills.ready_to_sync`, `payments.updated`, `transactions.declined`, and `users.invite_accepted` event types for webhook subscriptions.

## March 30, 2026

- **New List Bank Accounts endpoint** ŌĆö Retrieve a paginated list of bank accounts associated with your business. Supports `start` and `page_size` query parameters for pagination. [View endpoint](/developer-api/v1/api/bank-accounts#get-developer-v1-bank-accounts)

- **New List Vendor Agreements endpoint (beta)** ŌĆö Query vendor agreements with extensive filtering options including date ranges, renewal status, contract owners, departments, total value, and custom record filters via `POST /developer/v1/vendors/agreements`. [View endpoint](/developer-api/v1/api/vendors#post-developer-v1-vendors-agreements)

- **Enhanced vendor filtering by timestamps** ŌĆö Added `to_created_at`, `from_updated_at`, and `to_updated_at` query parameters to the List Vendors endpoint for filtering vendors by creation and update dates. [View endpoint](/developer-api/v1/api/vendors#get-developer-v1-vendors)

- **Vendor subsidiary data** ŌĆö Added `include_subsidiary` query parameter to the List Vendors endpoint. When set to `true`, vendor responses include a `subsidiary` field with ERP subsidiary identifiers. [View endpoint](/developer-api/v1/api/vendors#get-developer-v1-vendors)

- **Allowed overage percent override on limits** ŌĆö The `allowed_overage_percent_override` field is now available when creating limits and is returned in limit responses. This decimal value (0ŌĆō100) lets you set a fund-level override for the allowed overage percent, independent of the spend program or business default. [View endpoint](/developer-api/v1/api/limits#post-developer-v1-limits-deferred)

- **Update custom accounting field options** ŌĆö New `PUT /developer/v1/accounting/field-options/{field_option_id}` endpoint to update existing custom accounting field options. [View endpoint](/developer-api/v1/api/accounting#put-developer-v1-accounting-field-options-field-option-id)

- **New audit log event types** ŌĆö Added `Generated renewal brief for contract`, `Spend allocation change request approval policy updated`, and `Vendor Network updates enabled` event types to the audit logs endpoint.

- **Updated token prefix format** ŌĆö Access tokens are now prefixed with `ramp_business_tok_`, `ramp_user_tok_`, `ramp_business_jwt_`, or `ramp_user_jwt_` instead of the previous `ramp_tok_` prefix. Refresh tokens use `ramp_business_tok_` or `ramp_user_tok_`.

- **Custom records reference columns relaxed** ŌĆö The `corresponding_column_name` field on reference column configurations is no longer required and defaults to `null`. It will be auto-generated if not provided, simplifying forward-only reference column setup.

- **Auto-match uploaded receipts without a transaction ID** ŌĆö The `POST /receipts` endpoint can now automatically match uploaded receipts to transactions without requiring a `transaction_id`. [View endpoint](/developer-api/v1/api/receipts#post-developer-v1-receipts)

## March 18, 2026

- **Decimal quantity support for bill line items** ŌĆö The `unit_quantity` field has been replaced with `quantity`, which now accepts decimal values (e.g., `2.25`) instead of only integers. `unit_quantity` is still accepted for backwards compatibility. The `unit_price` field also now accepts decimal string or number formats. This applies when creating or updating line items on bills and draft bills. [View endpoint](/developer-api/v1/api/bills#post-developer-v1-bills)

- **New GL account classification types** ŌĆö Added six new enum values for GL account classifications: `AR_ACCOUNTS_RECEIVABLE_ACCOUNT`, `AR_BAD_DEBT_ACCOUNT`, `AR_CARD_FEES_ACCOUNT`, `AR_CASH_ACCOUNT_DEFAULT`, `AR_INCOME_ACCOUNT_DEFAULT`, and `AR_UNDEPOSITED_FUNDS_ACCOUNT`, enabling accounts receivable accounting integrations.
